Key Concepts

  • Tokens - The atomic units of LLM processing (‚âà4 chars per token in English)
  • Context Window - How much the model can "see" at once (8K ‚Üí 200K+ tokens)
  • Temperature - Creativity vs. determinism (0.0 = focused, 1.0 = creative)
  • Attention - How models weigh relevance across the context
üí° Pro Tip: Think of LLMs as incredibly sophisticated autocomplete that has read most of the internet. They predict the next token based on everything before it.

Mental Models to Adopt

  1. LLMs are stochastic, not deterministic - Same input can yield different outputs
  2. They don't "know" things - They recognize patterns and predict likely continuations
  3. Context is king - Better context = better outputs
  4. They're trained on human text - They inherit human biases and errors

The Prompt Engineering Framework

  1. Role - Who is the AI? (e.g., "You are a senior Python developer...")
  2. Context - What background does it need?
  3. Task - What specifically should it do?
  4. Format - How should the output be structured?
  5. Constraints - What should it avoid?
# Example: Well-Structured Prompt

## Role
You are a senior technical writer specializing in API documentation.

## Context  
I'm documenting a REST API for a payment processing system.
The audience is developers with 2-5 years of experience.

## Task
Write documentation for the following endpoint:
POST /api/v1/payments

## Format
- Use markdown
- Include: description, parameters table, example request/response, error codes
- Keep language concise and technical

## Constraints
- No marketing language
- Assume the reader knows HTTP basics

Prompt Patterns

  • Chain of Thought - "Think step by step before answering"
  • Few-Shot Learning - Provide examples of desired input/output
  • Self-Consistency - "Generate 3 solutions, then pick the best"
  • Metacognition - "Rate your confidence 1-10 and explain why"

Building Prompt Templates

# Email Response Template

## Context
I'm a [YOUR ROLE] at [COMPANY]. My communication style is [professional/casual/technical].

## The email I received:
"""
[PASTE EMAIL]
"""

## Task
Draft a response that:
- Addresses all points raised
- Maintains [TONE]
- Is concise (under 150 words)
- Suggests next steps if appropriate

## Format
- Start with a brief acknowledgment
- Address each point
- End with clear next action
üí° Workflow Pattern: Build a library of templates for your repetitive tasks. Store them in a notes app or text file for quick access.
