What is Vibe Coding?

Coined by Andrej Karpathy, vibe coding is:

"You fully give in to the vibes, embrace exponentials, and forget that the code even exists... It's not really coding ‚Äì I just see stuff, say stuff, run stuff, and copy paste stuff, and it mostly works."

The Spectrum of AI-Assisted Development

Level Description Tools When to Use
Pure Vibe AI writes everything, you describe intent Cursor Agent, Claude artifacts Prototypes, throwaway code, learning
AI-Heavy AI generates, you review and guide Cursor, Copilot, Claude Code Internal tools, MVPs, scripts
AI-Assisted You write, AI completes and suggests Copilot, Codeium, TabNine Production code, team projects
AI-Enhanced You write, AI reviews and explains Code review tools, docs generators Critical systems, regulated industries

The Vibe Coding Mindset

  1. Start with intent, not implementation - Describe what you want, not how to build it
  2. Iterate rapidly - Generate, test, refine, repeat
  3. Trust but verify - AI code works ~80% of the time; catch the 20%
  4. Learn the patterns - Understand what AI generates so you can debug it
  5. Know when to take over - Complex logic, security, performance = human oversight
üí° Key Insight: Vibe coding isn't about being lazy‚Äîit's about operating at a higher level of abstraction. You're the architect, AI is the contractor.

Cursor Core Features

  • ‚åòK (Cmd+K) - Inline edit: select code, describe change
  • ‚åòL (Cmd+L) - Chat: ask questions, get explanations
  • ‚åòI (Cmd+I) - Composer: multi-file generation and editing
  • Tab - Accept AI suggestions (like Copilot)
  • @ References - Include files, docs, or web in context

Cursor Workflow Patterns

# Pattern 1: Describe ‚Üí Generate ‚Üí Refine
1. Open Composer (‚åòI)
2. Describe: "Create a REST API endpoint for user registration 
   with email verification using Express and PostgreSQL"
3. Review generated files
4. Refine: "Add rate limiting and input validation"

# Pattern 2: Select ‚Üí Transform
1. Select existing code
2. Press ‚åòK
3. Describe change: "Refactor to use async/await instead of callbacks"
4. Review diff, accept or modify

# Pattern 3: Chat ‚Üí Understand ‚Üí Apply
1. Open Chat (‚åòL)
2. Ask: "Explain this function and suggest optimizations" + paste code
3. Apply suggestions via Composer

@ References Power Features

  • @filename - Include specific file in context
  • @folder - Include entire folder
  • @web - Search and include web results
  • @docs - Reference official documentation
  • @codebase - Search your entire codebase

GitHub Copilot

Best for: Real-time code completion, boilerplate, patterns

  • Tab - Accept suggestion
  • Alt+] / Alt+[ - Cycle through suggestions
  • Ctrl+Enter - Open Copilot completions panel
  • Comments drive suggestions - Write intent, get implementation

// and returns { valid: boolean, reason?: string }

  // Copilot will generate the implementation...

Claude Code (Terminal AI)

Best for: Complex multi-file changes, reasoning about architecture

# Install
npm install -g @anthropic-ai/claude-code

# Usage
claude-code

# Example prompts:
> "Refactor the authentication module to use JWT instead of sessions"
> "Add comprehensive error handling to all API endpoints"
> "Create unit tests for the user service"

Aider

Best for: Git-aware editing, pair programming with AI

# Install
pip install aider-chat

# Usage (in a git repo)
aider

# Features
- Automatically creates git commits
- Shows diffs before applying
- Supports multiple files
- Voice input available

# Example
> /add src/api/*.js
> "Add input validation to all POST endpoints"

Tool Selection Guide

Task Best Tool Why
Writing new code line-by-line Copilot Fastest inline completion
Multi-file feature implementation Cursor Composer Visual, handles file creation
Large refactoring Claude Code / Aider Better reasoning, git-aware
Quick code transformation Cursor ‚åòK Select ‚Üí describe ‚Üí done
Understanding existing code Cursor Chat / Claude Conversational explanation

üî® Build an App: Task Manager API (16:45 - 18:00)

Objective

Build a complete REST API for a task manager using AI-assisted development.

Requirements
  • CRUD operations for tasks (create, read, update, delete)
  • Task fields: id, title, description, status, due_date, created_at
  • Status values: todo, in_progress, done
  • Filter tasks by status
  • Basic input validation
  • Use any language/framework (Node/Express, Python/FastAPI, etc.)
Process (Use AI Throughout!)
  1. Scaffold (15 min) - Use Cursor Composer to generate project structure
  2. Implement (30 min) - Build endpoints with Copilot/Cursor assistance
  3. Test (15 min) - Generate and run tests with AI
  4. Document (10 min) - Auto-generate API documentation
  5. Review (5 min) - AI security/quality review
Starter Prompt (Cursor Composer)
Create a REST API for a task manager with:
- Express.js with TypeScript
- In-memory storage (array)
- Endpoints: GET /tasks, POST /tasks, GET /tasks/:id, PUT /tasks/:id, DELETE /tasks/:id
- Query param: ?status=todo|in_progress|done
- Input validation with express-validator
- Error handling middleware
- CORS enabled

Create: src/index.ts, src/routes/tasks.ts, src/types.ts, package.json, tsconfig.json
Deliverables
