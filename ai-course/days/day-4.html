What is Clawdbot?

Clawdbot is an AI assistant framework that lets you create personalized, persistent, tool-using AI assistants that connect to your existing tools and workflows.

Key Differences from ChatGPT/Claude

Feature ChatGPT/Claude Clawdbot
Personality Generic assistant Fully customizable (SOUL.md)
Context Session-based Persistent workspace files
Tools Limited plugins Full system access, exec, browser, etc.
Memory Basic memory feature Vector search + file-based memory
Channels Web/app only Slack, Telegram, Discord, Signal, etc.
Automation Manual interaction Heartbeats, cron, webhooks
Self-hosted No Yes (your machine, your data)

The Workspace Philosophy

Your AI assistant lives in a workspace â€” a directory on your machine. Key files:

  • SOUL.md - Who is this AI? Personality, values, communication style
  • USER.md - Who are you? Background, preferences, context
  • AGENTS.md - Operating instructions and behaviors
  • MEMORY.md - Long-term curated memories
  • memory/*.md - Daily logs and notes
  • TOOLS.md - Local tool configuration (API keys, paths, etc.)
ğŸ’¡ Philosophy: Your AI wakes up fresh each session. These files ARE its memory and personality. Write them well, and you get a capable, personalized assistant.

10:45 - 12:15 Installation & Configuration

ğŸ¯ By the end of this session, you will:

  • Have Clawdbot installed and running
  • Configure authentication with Anthropic
  • Create your workspace and core files
  • Test basic interaction via webchat

Prerequisites

  • Node.js 18+ installed
  • Anthropic API key (get one at console.anthropic.com)
  • macOS, Linux, or Windows with WSL

Installation

# Install Clawdbot globally
npm install -g clawdbot

# Verify installation
clawdbot --version

# Run the setup wizard
clawdbot wizard

# Start the gateway
clawdbot gateway start

# Open webchat (default: http://localhost:18789)

Configuration Wizard

The wizard will guide you through:

  1. Authentication - Connect your Anthropic API key
  2. Workspace - Set your working directory
  3. Model - Choose your default model (Claude Sonnet recommended)
  4. Optional channels - Telegram, Slack, etc.

Create Your Workspace

# Create a workspace directory
mkdir ~/ai-assistant
cd ~/ai-assistant

# Initialize with core files
cat > SOUL.md << 'EOF'
# SOUL.md - Who I Am

I am [YOUR ASSISTANT NAME], a professional AI assistant.

## Core Identity
- [Describe personality]
- [Communication style]
- [Key values]

## Operating Principles
- Be helpful and efficient
- Ask clarifying questions when needed
- Produce actionable outputs
EOF

cat > USER.md << 'EOF'
# USER.md - About My Human

- **Name:** [Your name]
- **Role:** [Your job/role]
- **Expertise:** [Your background]
- **Preferences:** [How you like to work]
EOF

cat > AGENTS.md << 'EOF'
# AGENTS.md - Operating Instructions

## Every Session
1. Read SOUL.md - this is who you are
2. Read USER.md - this is who you're helping
3. Check memory/ for recent context

## Memory
- Write to memory/YYYY-MM-DD.md for daily logs
- Update MEMORY.md for long-term memories

## Boundaries
- Ask before external actions (emails, posts)
- Keep private information private
EOF

Memory System

Clawdbot has a two-tier memory system:

  1. File-based memory - MEMORY.md and memory/*.md files loaded into context
  2. Vector search - Semantic search across memory files for relevant context
# Enable memory search in config
clawdbot config set agents.defaults.memorySearch.enabled true
clawdbot config set agents.defaults.memorySearch.provider openai

# Memory is now searchable
# Your assistant will use memory_search tool to find relevant context

Skills

Skills extend your assistant's capabilities:

# List available skills
clawdbot skills list

# Install a skill
clawdbot skills install weather
clawdbot skills install github

# Skills provide new tools and instructions
# The assistant automatically loads SKILL.md when relevant

Popular Skills

  • weather - Get weather forecasts
  • github - Interact with GitHub (issues, PRs, etc.)
  • notion - Manage Notion pages and databases
  • slack - Extended Slack operations
  • bird - Read X/Twitter via CLI

Sub-agents

Spawn background agents for long-running tasks:

# In conversation, your assistant can spawn sub-agents:
"I'll spawn a sub-agent to research this and ping you when done."

# The sub-agent runs in the background
# Notifies you via message when complete
# Useful for: research, building things, monitoring

Heartbeats

Make your assistant proactive:

# Configure heartbeat in config
clawdbot config set agents.defaults.heartbeat.every "1h"

# Create HEARTBEAT.md with periodic tasks
cat > HEARTBEAT.md << 'EOF'
# Heartbeat Tasks
- Check for unread important emails
- Review calendar for upcoming events
- Summarize recent Slack activity
EOF

# Your assistant wakes up hourly to check on things

Cron Jobs

# Schedule one-time or recurring tasks
clawdbot cron add --schedule "0 9 * * 1" --text "Generate weekly summary report"

# Reminders
clawdbot cron add --in "30m" --text "Remind Tim about the meeting"

Available Channels

  • Webchat - Built-in, always available
  • Slack - Socket Mode (easy) or HTTP
  • Telegram - Bot API integration
  • Discord - Bot integration
  • Signal - Via signal-cli
  • iMessage - macOS only
  • WhatsApp - Via Baileys

Slack Setup

# 1. Create a Slack App at api.slack.com/apps
# 2. Enable Socket Mode
# 3. Add Bot Token Scopes: chat:write, app_mentions:read, im:history, etc.
# 4. Install to workspace

# Configure Clawdbot
clawdbot channels configure slack

# Add tokens when prompted:
# - Bot Token (xoxb-...)
# - App Token (xapp-...)

# Start gateway with Slack enabled
clawdbot gateway restart

Telegram Setup

# 1. Create a bot with @BotFather on Telegram
# 2. Get the bot token

# Configure
clawdbot channels configure telegram

# Enter bot token when prompted

# Restart gateway
clawdbot gateway restart

# Message your bot on Telegram to pair

Access Controls

# Control who can talk to your bot

# DM Policy: pairing (invite code), allowlist, or open
clawdbot config set channels.telegram.dmPolicy pairing

# Group Policy: allowlist specific groups
clawdbot config set channels.slack.groupPolicy allowlist
clawdbot config set channels.slack.channels.C123456.enabled true

# Require mentions in group chats
clawdbot config set channels.slack.requireMention true

Multi-Channel Workflows

Your assistant can:

  • Receive a message on Slack, respond there
  • Proactively notify you on Telegram about something it found
  • Cross-reference conversations across channels
  • Route different tasks to different channels
